# Project Instructions

This project uses Ignite UI for Angular. Always follow these guidelines:

## Overview

This is a **DocFX-based documentation site builder** for [Ignite UI for Angular](https://github.com/IgniteUI/igniteui-angular) — a complete set of Material-based UI components for Angular by Infragistics. The documentation site supports three language versions: **English** (`en/`), **Japanese** (`jp/`), and **Korean** (`kr/`).

## Project Structure

```
/
├── en/                        # English documentation
│   ├── components/            # Component documentation (.md files)
│   │   ├── grid/              # Data Grid docs
│   │   ├── treegrid/          # Tree Grid docs
│   │   ├── hierarchicalgrid/  # Hierarchical Grid docs
│   │   ├── pivotGrid/         # Pivot Grid docs
│   │   ├── charts/            # Charts docs
│   │   ├── general/           # Getting started, guides
│   │   ├── themes/            # Theming docs
│   │   └── grids_templates/   # Shared grid topic templates
│   ├── docfx.json             # DocFX build configuration
│   ├── global.json            # Global metadata
│   └── toc.yml                # Table of contents
├── jp/                        # Japanese documentation (same structure)
├── kr/                        # Korean documentation (same structure)
├── gulpfile.js                # Gulp build tasks
├── package.json               # Node.js dependencies and npm scripts
├── .markdownlint.json         # Markdown linting rules
└── cspell.json                # Spell-check configuration
```

## Development Commands

```bash
# Install dependencies (also restores dotnet tools via postinstall)
npm install

# Start development server
npm start -- --lang en   # English
npm start -- --lang jp   # Japanese
npm start -- --lang kr   # Korean

# Build static site
npm run build -- --lang en

# Lint markdown files
npm run lint:md

# Spell check
npm run spellcheck

# Run both checks
npm run verify
```

## Documentation Conventions

### Front Matter

All component documentation files must include YAML front matter:

```markdown
---
title: Angular Component Name | Brief description | Infragistics
_description: Detailed description for SEO.
_keywords: keyword1, keyword2, ignite ui for angular
_license: commercial
---
```

### Heading Style

- Use ATX-style headings (`#`, `##`, `###`)
- Increment heading levels by one at a time only (no skipping levels)
- Single `#` for the page title
- `##` for main sections, `###` for sub-sections

### Code Blocks

Always use fenced code blocks with a language identifier:

```markdown
    ```typescript
    // TypeScript code here
    ```
    ```html
    <!-- HTML template here -->
    ```
    ```scss
    // SCSS styles here
    ```
```

### Live Sample Embeds

Use `<code-view>` to embed interactive samples:

```html
<code-view style="height: 400px"
           data-demos-base-url="{environment:demosBaseUrl}"
           iframe-src="{environment:demosBaseUrl}/grids/grid/basic-usage"
           alt="Angular Grid Example">
</code-view>
```

Optionally set the live editor explicitly:

```html
<code-view explicit-editor="stackblitz" ... ></code-view>
<code-view explicit-editor="csb" ... ></code-view>
```

### Callouts / Alerts

```markdown
>[!NOTE]
> This is a note.

>[!WARNING]
> This is a warning.

>[!IMPORTANT]
> This is important.
```

### Collapsible Snippets

```html
<div class="fancy-details">
    <details>
        <summary>Summary text</summary>
        <code>// code here</code>
    </details>
</div>
```

### Internal Links

Use relative paths:

```markdown
[Getting Started](general/getting-started.md)
[Themes](../themes.md)
```

### Images

Always include alt text (MD045 is enforced):

```markdown
![Description of image](../../images/general/example.png)
```

### Markdown Linting

Key enforced rules (`.markdownlint.json`):

- **MD001**: Headings increment one level at a time
- **MD003**: ATX heading style (`#`)
- **MD004**: Unordered lists use dashes (`-`)
- **MD007**: Unordered list indentation: 2 spaces
- **MD046**: Fenced code block style
- **MD048**: Backtick code fences
- **MD049**: Emphasis with underscores (`_italic_`)
- **MD050**: Strong with asterisks (`**bold**`)

Disabled: MD013 (line length), MD033 (inline HTML), MD040 (code fence language), MD041 (first line heading).

## Table of Contents Labels

In `[lang]/components/toc.yml`, you can annotate topics with optional labels:

```yaml
- name: Component Name
  href: component-name.md
  new: true      # Shows a "new" badge
  updated: true  # Shows an "updated" badge
  preview: true  # Shows a "preview" badge
  beta: true     # Shows a "beta" badge
```

## Grid Templates

Many grid documentation topics are shared via `grids_templates/` and generated by Gulp tasks for:

- `IgxGridComponent` → `en/components/grid/`
- `IgxTreeGridComponent` → `en/components/treegrid/`
- `IgxHierarchicalGridComponent` → `en/components/hierarchicalgrid/`
- `IgxPivotGridComponent` → `en/components/pivotGrid/`

When modifying shared grid topics, edit the template file in `grids_templates/`, not the generated file.

## Environment

Set `NODE_ENV` in a `.env` file at the project root:

```
NODE_ENV=development   # local dev
NODE_ENV=staging       # staging build
NODE_ENV=production    # production build
```
